Q161. Calculate the time complexity of the following program. 
void printUnorderedPairs(int[] arrayA, int[] arrayB){
    for(int i = 0; i < arrayA.length; i++){
        for(int j = 0; j < arrayB.length; j++){
            if(arrayA[i] < arrayB[j]){
                System.out.println(arrayA[i] + "," + arrayB[j]);
            }
        }
    }
}

  • Outer loop (for i) runs A times (once for each element in arrayA).
  • Inner loop (for j) runs B times (once for each element in arrayB).
  • For each iteration of i, the inner loop runs B times.
  • The if condition and System.out.println are O(1) operations per iteration.
  • Total Time Complexity: The nested loops result in A √ó B iterations.
  • Thus, the time complexity is O(A √ó B).
Q162. What do these expressions evaluate?
  • 1. true && false
  • 2. true && false || true

  • && (AND) is stricter than || (OR):
    • Needs both sides to be true to return true.
  • Operator Precedence Matters:
    • && is evaluated before || unless parentheses are used.
    • Example: true || false && false ‚Üí true || (false && false) ‚Üí true || false ‚Üí true.
  • Short-Circuiting:
    • Java stops evaluating && if the left operand is false (since the result is already determined).
    • Similarly, || stops if the left operand is true.

Q167. Calculate the space complexity of the following program. 
void createArray(int n) {
    int[] arr = new int[n];
    for (int i = 0; i < n; i++) {
        arr[i] = i * 2;
    }
}

  • int[] arr = new int[n]; ‚Üí This dynamically allocates an array of size n.
  • The space taken by the array grows linearly with n.
  • Additional variables (i):
    • The loop variable i uses constant space O(1), which does not affect the overall space complexity.
  • Total Space Complexity:
    • Dominated by the array allocation ‚Üí O(N).

Q171. What will be the output of the following Java code snippet? 
class abc
{
    public static void main(String args[])
    {
        if(args.length > 0)
        System.out.println(args.length);
    }
}

  • args.length checks the number of command-line arguments passed to the program.
  • If no arguments are provided, args.length is 0, so the condition args.length > 0 evaluates to false.
  • Since the condition is false, the System.out.println statement is not executed.
  • Thus, no output is printed.
