Q81. What phrase indicates that a function receives a copy of each argument passed to it rather than a reference to the objects themselves?

  • Java is strictly pass-by-value, even for object references.
  • This means that copies of variables are passed to methods.
  • For primitive types, the value itself is copied.
  • For objects, the reference is copied, not the object itself.
Q82. In Java, what is the scope of a method's argument or parameter?

  • Method parameters are local variables.
  • They are accessible only within the method in which they are declared.
  • Once the method ends, the parameters go out of scope.
Q83. What is the output of this code? 
public class Main {
  public static void main (String[] args) {
    int[] sampleNumbers = {8, 5, 3, 1};
    System.out.println(sampleNumbers[2]);
  }
}

  • Java arrays are zero-indexed.
  • Index 2 refers to the third element.
  • The array is {8, 5, 3, 1} — so index 2 is 3.

Q85. What is the output of this code? 
import java.util.*;
class Main {
  public static void main(String[] args) {
    String[] array = new String[]{"A", "B", "C"};
    List<String> list1 = Arrays.asList(array);
    List<String> list2 = new ArrayList<>(Arrays.asList(array));
    List<String> list3 = new ArrayList<>(Arrays.asList("A", new String("B"), "C"));
    System.out.print(list1.equals(list2));
    System.out.print(list1.equals(list3));
  }
}

  • equals checks content equality, not reference.
  • All lists contain the same elements in the same order.
  • Even though new String("B") creates a new object, equals still returns true due to value match.
Q86. What is the output of this code? 
public class Main {
  public static void main(String[] args) {
    String a = "Hello";
    String b = "Hello";
    System.out.print(a == b);
    System.out.print(" ");
    System.out.print(a.equals(b));
  }
}

  • String literals are stored in the String pool.
  • Variables a and b point to the same object.
  • == returns true for same reference; equals() returns true for same value.
Q87. What is the main purpose of the `final` keyword in Java?

  • final prevents modification:
  • Final variable – value cannot be reassigned.
  • Final method – cannot be overridden.
  • Final class – cannot be subclassed.
Q88. What will be the output? 
public class Main {
  public static void main(String[] args) {
    String s1 = new String("Hello");
    String s2 = new String("Hello");
    System.out.println(s1 == s2);
  }
}

  • new String("Hello") creates a new object.
  • s1 == s2 compares references, not values.
  • They are different objects → result is false.
Q89. Which data structure uses LIFO (Last In First Out) ordering?

  • Stack is based on Last-In-First-Out (LIFO).
  • The most recently added element is removed first.
  • Queue is FIFO (First-In-First-Out), opposite of stack.
Q90. What is the default value of a boolean variable in Java?

  • In Java, boolean instance variables default to false.
  • Only local variables are not automatically initialized.
  • Default values are set for class-level fields.
