Q101. Which statement about constructors is not true?

  • A class can have multiple constructors with different parameters (constructor overloading).
  • Constructors can call other constructors using this() or the superclass constructor with super().
  • Constructors do not have return types, not even void.
  • If no constructor is defined, Java provides a default no-argument constructor automatically.
  • Therefore, it is not true that every class must explicitly define a no-argument constructor.
Q102. What language feature allows types to be parameters on classes, interfaces, and methods in order to reuse the same code for different data types?

  • Generics allow classes, interfaces, and methods to operate on types specified as parameters, improving code reuse and type safety.
  • Regular Expressions and Reflection are unrelated to parameterizing types.
  • Concurrency relates to multithreading, not type parameters.
Q103. What will be printed? 
public class Berries{

    String berry = "blue";

    public static void main(String[] args) {
        new Berries().juicy("straw");
    }
    void juicy(String berry){
        this.berry = "rasp";
        System.out.println(berry + "berry");
    }
}

  • The method parameter berry shadows the instance variable berry.
  • this.berry = "rasp"; updates the instance variable to "rasp".
  • But System.out.println(berry + "berry"); prints the method parameter "straw" concatenated with "berry".
  • Thus, output is strawberry.
Q104. What is the value of forestCount after this code executes? 
Map<String, Integer> forestSpecies = new HashMap<>();

forestSpecies.put("Amazon", 30000);
forestSpecies.put("Congo", 10000);
forestSpecies.put("Daintree", 15000);
forestSpecies.put("Amazon", 40000);

int forestCount = forestSpecies.size();

  • HashMap keys are unique.
  • Adding forestSpecies.put("Amazon", 40000); replaces the previous "Amazon" entry (30000).
  • So, the final map has 3 entries: "Amazon", "Congo", "Daintree".
  • Therefore, forestCount = 3.
Q105. What is the problem with this code? 
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;

class Main {

    public static void main(String[] args) {
        List<String> list = new ArrayList<>(Arrays.asList("a", "b", "c"));
        for(String value : list) {
            if(value.equals("a")) {
                list.remove(value);
            }
        }
        System.out.println(list); // outputs [b,c]
    }
}

  • Removing items from a collection while iterating over it using a for-each loop causes a ConcurrentModificationException.
  • String comparison using equals() is correct here.
  • ArrayList implements List, so that is not an issue.
Q106. How do you convert this method into a lambda expression? 
public int square(int x) {
    return x * x;
}

  • Lambda expressions use the syntax: (parameters) -> expression.
  • If the lambda body is a single expression, braces and return statement can be omitted.
  • The correct lambda matching the method is x -> x * x.
  • The other options either have syntax errors or missing parameters.

Q108. What is the result of this program? 
interface Foo {
    int x = 10;
}

public class Main{

    public static void main(String[] args) {
        Foo.x = 20;
        System.out.println(Foo.x);
    }
}

  • Interface variables are implicitly public static final.
  • You cannot assign a new value to a final variable after it is initialized.
  • Therefore, Foo.x = 20; causes a compile-time error.
Q109. Which statement must be inserted on line 1 to print the value true?

  • Optional.of(val) throws NullPointerException if val is null.

Q110. What will this code print, assuming it is inside the main method of a class? 
System.out.println(true && false || true);
System.out.println(false || false && true);

  • Logical AND (&&) has higher precedence than logical OR (||).
  • First line: true && false || true evaluates as (true && false) || true → false || true → true.
  • Second line: false || false && true evaluates as false || (false && true) → false || false → false.
  • Therefore, the output is:
  • true and then false.
